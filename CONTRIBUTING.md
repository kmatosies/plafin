# Contribuindo com o Plafin

Obrigado por considerar contribuir com o projeto.

## Como contribuir

1. Abra uma issue descrevendo bug, melhoria ou proposta técnica.
2. Crie uma branch com escopo claro (`feat/...`, `fix/...`, `docs/...`).
3. Mantenha commits pequenos e objetivos.
4. Abra um Pull Request com contexto, impacto e checklist de testes.

## Padrões importantes

- Nunca commite segredos, chaves, tokens ou arquivos `.env`.
- Prefira variáveis de ambiente e placeholders seguros.
- Atualize documentação quando alterar fluxo, setup ou comportamento.

## Desenvolvimento local

- Backend: `backend/README.md`
- Frontend: `frontend/README.md`
- Deploy/infra: `docs/deploy.md`

## Checklist de PR

- [ ] Sem credenciais ou dados sensíveis no diff
- [ ] Mudança documentada quando necessário
- [ ] Testes locais executados (quando aplicável)
- [ ] Sem breaking changes não documentadas
